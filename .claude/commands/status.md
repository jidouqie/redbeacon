---
description: 查看账号运营状态 — 所有账号概览 + 单账号详情，内容数据、选题库、调度器状态一览
---

> **【系统状态总览】** 一键查看全系统健康状况：所有账号登录状态速览、选题库余量、待审核内容数、调度器运行状态。**出问题不知道从哪查时先跑这个。**

你是 RedBeacon 的运营助手。后端运行在 `http://localhost:8000`。

---

## 第一步：获取所有账号

```bash
curl -s http://localhost:8000/api/accounts
curl -s http://localhost:8000/api/automation/status
```

---

## 第二步：账号概览表（多账号时优先展示）

无论账号数量多少，先用表格展示所有账号的状态速览：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📊 RedBeacon 账号总览
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

| # | 账号名 | 小红书登录 | 自动生成 |
|---|---|---|---|
| 1 | 账号A | ✓ 已登录 | 开启 |
| 2 | 账号B | ✗ 未登录 | 关闭 |
```

账号名取 `display_name`，若为空则取 `nickname`，再为空则显示"账号 {id}"

> 注：MCP 是按需启动的，发布/验证登录时自动管理，平时为"已停止"是正常状态。

---

## 第三步：逐账号详情

对每个账号，并行获取详情：

```bash
# 对每个 ACCOUNT_ID 执行：
curl -s http://localhost:8000/api/strategy/{ACCOUNT_ID}
curl -s http://localhost:8000/api/topics/{ACCOUNT_ID}/stats
curl -s "http://localhost:8000/api/content/{ACCOUNT_ID}?status=pending_review&limit=5"
```

展示格式（每个账号一块）：

```
【账号名】
登录状态：已登录 / 未登录
定位：[niche，未配置则提示"未配置，运行 /onboard 开始"]

选题库：总计 N 条 | 未用 N 条 | 已用 N 条
  [如未用 < 5，提示"选题库即将耗尽，建议补充"]

待审核内容：N 篇
  [列出标题，最多 3 条]
```

---

## 第四步：全局状态 + 行动建议

```
【全局调度器】
状态：运行中 / 已停止
任务：[下次执行时间]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

根据状态给出 **最多 3 条**行动建议，按以下优先级选取最紧迫的：

| 优先级 | 条件 | 建议 |
|---|---|---|
| 最高 | 有账号未登录 | `/mcp login` 扫码授权，否则无法发布 |
| 高 | 有账号 `niche` 为空 | `/onboard` 梳理定位，AI 生成没有方向 |
| 高 | 有待审核内容 | `/review` 审核内容，审核通过才能发布 |
| 中 | 有 approved 内容但飞书未配置 | `/setup-feishu` 配置飞书后才能推送发布 |
| 中 | 某账号选题库未用 < 5 条 | `/topics` 补充选题（指出是哪个账号）|
| 低 | 飞书未配置且无 approved 内容 | 提示发布流程依赖飞书，可提前配置 |

同时在建议末尾附一行内容状态说明：

```
【内容流转】pending_review（待审核）→ approved（已通过）→ published（已发布）
当前：[各状态数量，如"待审核 3 篇 / 已通过 1 篇 / 已发布 12 篇"]
```
